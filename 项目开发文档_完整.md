## 1. 项目概述

### 1.1 项目背景

随着播客行业的快速发展，越来越多的创作者加入到播客创作的行列。然而，传统的音频剪辑工具学习门槛高、操作复杂，需要专业技能才能完成高质量的播客制作。特别是对于内容创作者来说，剪辑音频往往需要花费大量时间在对齐音轨、识别口癖等机械劳动上，而无法专注于内容创作本身。

同时，AI技术的快速发展为音频内容创作领域带来了新的可能性。如何将AI技术与播客创作相结合，降低创作门槛，提高创作效率，成为当前亟待解决的问题。

### 1.2 项目目标

PodPal旨在打造一个一站式的智能播客创作平台，通过AI技术赋能内容创作者，实现：

- **降低创作门槛**：让创作者像编辑Word文档一样编辑音频
- **提高创作效率**：利用AI自动化处理剪辑、转写、优化等工作
- **丰富创作内容**：提供智能生成、内容增值等工具
- **简化发布流程**：支持一键分发到多个平台

### 1.3 项目意义

- **对创作者**：将创作者从繁琐的技术工作中解放出来，专注于内容创作
- **对行业**：推动播客创作的普及和发展，降低行业准入门槛
- **对技术**：探索AI在音频内容创作领域的应用，推动相关技术进步

#### 4.1.2 智能口癖识别

**实现思路：**
1. 定义口癖词列表（"就是"、"然后"、"那个"、"嗯"、"啊"、"呃"、"诶"等）
2. 使用正则表达式进行分词
3. 匹配口癖词并高亮标注
4. 改进口吃识别算法，避免误识别"冷冰冰"等正常词汇
5. 只标注不删除，由用户自行决定是否删除

**关键代码（移动端）：**
```dart
// edit_step.dart
final List<String> _fillerWords = ['就是', '然后', '那个', '嗯', '啊', '呃', '诶', '你知道', '对吧', '可以说', '怎么说呢', '就是说'];

// 改进的口吃匹配：匹配连续重复的单字，至少3个相同字符
final stutterRegex = RegExp(r'([\u4e00-\u9fa5A-Za-z])\1{2,}');

List<Map<String, dynamic>> _parseTranscriptToTokens(String text) {
  final List<Map<String, dynamic>> tokens = [];
  
  // 正则匹配：中文单词、英文单词、数字、标点符号、空白字符
  final pattern = RegExp(r'([\u4e00-\u9fa5]+|[a-zA-Z0-9]+|[^\w\s\u4e00-\u9fa5]+|[\s]+)');
  
  final matches = pattern.allMatches(text);
  for (final match in matches) {
    final part = match.group(0)!;
    if (RegExp(r'^\s+$').hasMatch(part)) {
      tokens.add({'type': 'space', 'text': part});
    } else if (RegExp(r'^[^\w\s\u4e00-\u9fa5]+$').hasMatch(part)) {
      tokens.add({'type': 'punctuation', 'text': part});
    } else {
      // 检查是否是口癖词
      if (_fillerWords.contains(part)) {
        tokens.add({'type': 'filler', 'text': part});
      } else {
        _addTokenToTokens(part, tokens);
      }
    }
  }
  
  return tokens;
}
```

- 智能降噪算法

#### 5.1.5 AI生成技术

- 大语言模型文本生成
- TTS文本转语音
- 音色克隆
- 视频自动生成

### 5.2 创新点

#### 5.2.1 文本化音频编辑

**创新点：** 将音频编辑转化为文本编辑，让用户像编辑Word文档一样编辑音频，大大降低了音频剪辑的学习门槛。

**技术实现：**
- ASR语音转文字，生成带时间戳的转写文本
- 建立文本片段与音频位置的映射关系
- 点击文本跳转到对应音频位置
- 删除文本片段即删除对应音频

#### 5.2.2 智能口癖识别与标注

**创新点：**
1. 改进的口吃识别算法，避免将"冷冰冰"等正常词汇误识别
2. 只高亮标注不自动删除，由用户自主决定
3. 完整的口癖词列表，覆盖"那个"、"诶"等常用词

**技术实现：**
- 至少3个相同字符才识别为口吃的正则表达式
- 基于分词的口癖词匹配
- 高亮显示而非直接修改

#### 5.2.3 沉浸式三步声演实验室

**创新点：** 将声演功能重构为三步引导式流程，降低操作复杂度，提升用户体验。

#### 5.2.4 灵活的插入位置调整

**创新点：** 支持通过滑块直观选择插入位置，从开头到结尾任意位置，提供良好的用户交互体验。

#### 5.2.5 多端协同创作

**创新点：** 同时提供Web端和移动端，用户可以在不同设备上继续创作，数据实时同步。
